# Changelog

All notable changes to Token-Goat Mem are documented in this file. **This file is the canonical version history** â€” `package.json` mirrors the latest release; if a version string anywhere disagrees with this file, this file wins. Format follows Keep a Changelog. Token-Goat Mem follows Semantic Versioning starting at 1.0.

## [Unreleased]

### Fixed

- **`mem init` crashed with a raw error (not the `WiringConflictError` contract) on some hand-edited configs.** Two more spots that assumed well-formed input reached uncaught type errors instead of the documented "refusing to modify a hand-edited config" abort: (1) a `hooks.SessionStart` array element that is `null`, a primitive, or an object whose `hooks` isn't an array (`installClaudeSettings` threw `TypeError: Cannot read properties of null`; `uninstallClaudeSettings` likewise crashed instead of leaving the file untouched); (2) a `tasks.json` whose `tasks` or `inputs` key is present but not an array (`installTasksJson` reached jsonc-parser's raw `Can not add property to parent of type string`). Both now abort with `WiringConflictError` on install and are left untouched on uninstall, matching how the root/`hooks`/`SessionStart`-array checks already behaved.
- **`mem import --dry-run` created the database it claimed not to write.** The dry-run still opened mem's SQLite store, which runs `mkdirSync` + creates the db file, WAL sidecars, and schema on disk â€” so previewing an import in a project with no store yet silently created `~/.mem/mem.db`. The dry-run now runs entirely from the markdown file (`planImportFromMarkdown`) and opens no database, honoring the documented "nothing written".
- **Secret screening let unlabeled high-entropy secrets past through `anchor` and `sourceRef`.** A prior 0.2.0-era fix for the `sourceRef`/`anchor` path false-positive (see 0.2.0 below) overcorrected: `anchor` was exempted from the generic high-entropy heuristic field-wide instead of per-token, and `sourceRef` was excluded from screening entirely â€” including `SECRET_PATTERNS`. Since `mem remember --source-ref <ref>` accepts an arbitrary user/agent-supplied string (not just the programmatic `<path>:<line>` pointer `mem import` produces), a real credential placed in either field could be stored with zero screening. Both fields are now screened like any other: named `SECRET_PATTERNS` always run, and only a slash-containing token skips the generic entropy fallback (still avoiding the original path false-positive).
- **`mem edit` bypassed every field-level guard `mem remember` enforces.** Editing a fact's `--text`, `--subject`/`--value`, or `--anchor` went straight to storage with no length/emptiness checks and no anchor-syntax validation, so `mem edit` could store an over-length or empty text, or a malformed anchor (e.g. an unknown predicate) that `mem remember` would reject outright â€” the malformed anchor then permanently evaluates as "unverified" freshness. `mem edit` now runs the same validation `captureExplicit` does before writing. Also: a secret blocked by `mem edit` now writes an `edit_blocked_secret` audit_log entry, matching the audit trail both capture paths already wrote for a blocked secret.
- **`mem init copilot-vscode` crashed with a raw error on a `keybindings.json` containing literally `null`.** `installKeybindings` coerced the parsed root with `null ?? []`, which let a JSON `null` masquerade as an empty array, slip past the "not a JSON array" check, and reach jsonc-parser's `modify(text, [-1], â€¦)` on a `null` root â€” throwing a raw `Can not add property to parent of type null` Error instead of the documented `WiringConflictError` contract. Every non-array root (`null`, number, string, boolean, object) now aborts with `WiringConflictError`, matching how `installTasksJson` rejects a non-object root. (This was the last JSON/JSONC wiring handler still carrying the coerce-instead-of-reject bug class.)
- **Contradiction detection collapsed same-subject facts from unrelated projects into one bucket.** `detectContradictions` keyed only on `subject` + `scope`, but mem's store is shared across every project, so `subject="package-manager" scope="project"` in project A (value `npm`) and the same subject/scope in project B (value `pnpm`) looked like a live contradiction. On the whole-store paths (`mem review`, `mem epoch --gc`, plain `mem recall`) this meant one project's fact was mislabeled â€” and, under `epoch --gc`, *persistently superseded* â€” because an unrelated project chose a different value. The bucket key now includes `scope_root` for non-global scopes (globals still share one bucket), so each project's facts stay independent. The `--hint-format` seam was already immune (it pre-filters to a single root before detection); this fixes every other caller.

## [0.2.0] - 2026-07-13

### Added

- **`mem init <tool>` / `mem uninstall <tool>`** â€” automates the wiring docs/integrations/*.md previously asked a human to hand-copy, for `claude-code`, `codex`, `copilot-cli`, and `copilot-vscode`. `init` writes each tool's documented config exactly (Claude Code: `SessionStart` hook in `.claude/settings.json` project or `~/.claude/settings.json` user via `--user`, plus `CLAUDE.md`; Codex/Copilot CLI: `AGENTS.md`; Copilot VS Code: `.vscode/tasks.json`, user `keybindings.json`, plus `AGENTS.md`); `uninstall` reverses exactly what `init` wrote and nothing else. `--dry-run` on either command previews every file it would touch without writing. `--all` on `uninstall` removes every tool's wiring in one call (a safe no-op on tools that were never installed).
- Idempotent by construction: re-running `init` upgrades mem's own entries in place rather than duplicating them. `CLAUDE.md` (written only by `claude-code`) uses a single-owner HTML-comment marker pair. `codex`, `copilot-cli`, and `copilot-vscode` all write into the same `AGENTS.md`, so instead of three near-duplicate blocks they share one reference-counted marker (`<!-- token-goat-mem:start tools=<sorted,deduped,csv> -->` / `<!-- token-goat-mem:end -->`) â€” `init` adds the installing tool to the sorted `tools=` list (creating the block on the first tool, joining it on each subsequent one), and `uninstall` drops it from that list, only removing the block once the last tracked tool uninstalls. JSON/JSONC files (`settings.json` hooks, VS Code `tasks.json`/`keybindings.json`) stamp every object mem writes with an inert `__token_goat_mem` key, which install upgrades/skips and uninstall strips â€” surviving content drift across mem versions. An unstamped entry already occupying the same identity (hook command, task label, keybinding key) aborts the write with a conflict error instead of duplicating or silently overwriting hand-written config.
- Every write is atomic (temp file + rename), takes a one-time `.bak` snapshot of a pre-existing file before its first write (never overwritten on a later re-init), and retries once against freshly-read content if the file changed underneath the write.
- New dependency: `jsonc-parser` (zero transitive deps), used to edit `tasks.json`/`keybindings.json` via its `modify()`/`applyEdits()` API so a user's existing comments and formatting survive.
- Published to npm; CI (`.github/workflows/publish.yml`) publishes on `v*` tag push or manual dispatch, gated on the full `lint`/`typecheck`/`test`/`build` suite via `prepublishOnly`.

### Fixed

- **Secret screening false-positived on `sourceRef`** â€” capture's entropy screen was scanning the internal `<resolved path>:<line>` provenance pointer alongside actual fact content. Its generic-token pattern's alphabet includes `/` (needed to catch real base64 secrets, which legitimately contain `/`), so any long forward-slash-delimited path â€” any Linux/macOS path over ~32 characters, including `mem import --from-md`'s own dedup key â€” could be blocked as a "high-entropy secret" from directory-name entropy alone, with no secret present. Windows paths never triggered this (`\` isn't in the token alphabet), which is how it stayed latent until Linux CI was wired up. `sourceRef` is no longer screened.
- **`copilot-vscode` wiring tests hardcoded a Windows-only path** â€” verified against `~/.config/Code/User` (Linux) and `~/Library/Application Support/Code/User` (macOS) in addition to `%AppData%/Roaming/Code/User` (Windows); the installed code itself already branched correctly, only the tests assumed one OS.

## [0.1.0] - 2026-07-13

Initial release: a local-first, correctness-focused long-term memory CLI for AI coding agents (Claude Code, Copilot CLI, Copilot in VS Code, Codex).

### Added

- **Eleven CLI commands** (`mem`, bundled to `dist/token-goat-mem.mjs`): `remember`, `import` (`--from-md`, advisory-only markdown bullet import), `recall`, `list`, `show`, `forget` (soft delete, kept for audit), `pin`, `edit`, `review` (with `--promote`/`--reject` for pending facts, `--summary`/`--section`/`--since-epoch` filters), `epoch` (with `--gc` retention pass: persists contradiction resolutions, prunes superseded facts/sources/audit rows, applies preference decay), and `doctor` (read-only environment/DB health check).
- **SQLite storage** at `~/.mem/mem.db` (override via `TOKEN_GOAT_MEM_HOME`), WAL mode, three tables (`facts`, `sources`, `audit_log` + `meta`), a monotonic write epoch (with a `facts.epoch` column for `--since-epoch` filtering), zero network calls.
- **Anchor-based staleness detection** â€” nine pure read-only filesystem/git predicates (`file-exists`, `file-absent`, `file-newer-than`, `file-contains`, `file-not-contains`, `newest-of`, `glob-exists`, `git-branch-is`, `git-tracked`) evaluated per recall with three-valued verdicts: `affirmed` / `unverified` / `contradicted`. Only `affirmed` is ground-truth eligible; contradicted facts are excluded from recall and routed to `mem review`. No shell-out (git predicates read `.git/HEAD` and `.git/index` directly), bounded I/O, paths confined to `--root`.
- **Deterministic contradiction resolution** â€” `subject`+`value` keying; same subject + scope with a different value marks the loser `superseded` (newer + higher-provenance wins), or `contested` and withheld from ground truth when genuinely ambiguous.
- **Secret screening on capture** â€” named secret patterns plus a generic high-entropy screen; refused facts name a `.mem/allowlist` escape hatch. Suggested (derived) capture, including markdown import, exists as a library seam that can only ever store `pending` facts.
- **BM25 recall with a correctness gate** â€” relevance ranking, then freshness re-validation, trust filtering, and contradiction/contested exclusion. `--stable` forces deterministic id-sorted output when reproducible ordering matters more than relevance/recency. `--hint-style terse` trims the CTA and shortens kind labels for space-constrained callers. An embedding backend is an injectable seam; no concrete backend ships in v1 (BM25-only in practice).
- **`mem import --from-md <path>`** â€” advisory-only import: parses a markdown file (CLAUDE.md-style) for `-`/`*` bullet-list lines that look like preference/decision-shaped statements and stores each as a **`pending`, `source_type: "derived"`** fact through the same `captureSuggested` trust path as any other suggested candidate â€” no code path writes `active`, and there is no bulk-promote shortcut; every imported fact still requires an explicit `mem review --promote <id>`. Bullets nested under an obviously structural heading (e.g. `Architecture`, `File Structure`) are skipped. Provenance (`<absolute file path>:<line>`) is recorded in the fact's `source_ref` field. `--dry-run` reports candidates without writing. Re-importing the same file skips bullets already imported at the same `source_ref` + text.
- **Token-goat seam** â€” `mem recall --hint-format --root <project-root>` emits the `TGMEM/2` wire format by default: per-fact `display` strings carry no per-line follow-up CTA; instead one shared footer line (`footer  mem show <id> for detail; mem review to resolve contested/pending`) is appended once, when at least one fact line was emitted. `TGMEM/1` (the original per-line-CTA, no-footer format) remains fully supported via `HintFormatOptions.protocolVersion: 1` on the programmatic seam (`buildHintFormat()`); the CLI itself always emits the current default. The seam is one-directional (Mem reads nothing from token-goat), stateless, and self-caveating.
- **Cheap polling via `mem epoch`** â€” a monotonic integer bumped by every write and left untouched otherwise, so a host tool can skip a full `mem recall` when nothing changed. Plain `mem epoch` never mutates state.
- **Test suite** â€” 232 vitest tests including fast I/O-free structural guards (`npm run test:guards`) and end-to-end tests against the shipped bundle, isolated from real user data via `TOKEN_GOAT_MEM_HOME`.
- **Docs** â€” README with a verified copy-paste walkthrough, integration guides for Claude Code / Copilot CLI / Copilot in VS Code / Codex, `AGENTS.md` + `CLAUDE.md`, `SECURITY.md`, `CONTRIBUTING.md`.
