## [0.2.0] - 2026-07-13

### Added

- **`mem init <tool>` / `mem uninstall <tool>`** — automates the wiring docs/integrations/*.md previously asked a human to hand-copy, for `claude-code`, `codex`, `copilot-cli`, and `copilot-vscode`. `init` writes each tool's documented config exactly (Claude Code: `SessionStart` hook in `.claude/settings.json` project or `~/.claude/settings.json` user via `--user`, plus `CLAUDE.md`; Codex/Copilot CLI: `AGENTS.md`; Copilot VS Code: `.vscode/tasks.json`, user `keybindings.json`, plus `AGENTS.md`); `uninstall` reverses exactly what `init` wrote and nothing else. `--dry-run` on either command previews every file it would touch without writing. `--all` on `uninstall` removes every tool's wiring in one call (a safe no-op on tools that were never installed).
- Idempotent by construction: re-running `init` upgrades mem's own entries in place rather than duplicating them. `CLAUDE.md` (written only by `claude-code`) uses a single-owner HTML-comment marker pair. `codex`, `copilot-cli`, and `copilot-vscode` all write into the same `AGENTS.md`, so instead of three near-duplicate blocks they share one reference-counted marker (`<!-- token-goat-mem:start tools=<sorted,deduped,csv> -->` / `<!-- token-goat-mem:end -->`) — `init` adds the installing tool to the sorted `tools=` list (creating the block on the first tool, joining it on each subsequent one), and `uninstall` drops it from that list, only removing the block once the last tracked tool uninstalls. JSON/JSONC files (`settings.json` hooks, VS Code `tasks.json`/`keybindings.json`) stamp every object mem writes with an inert `__token_goat_mem` key, which install upgrades/skips and uninstall strips — surviving content drift across mem versions. An unstamped entry already occupying the same identity (hook command, task label, keybinding key) aborts the write with a conflict error instead of duplicating or silently overwriting hand-written config.
- Every write is atomic (temp file + rename), takes a one-time `.bak` snapshot of a pre-existing file before its first write (never overwritten on a later re-init), and retries once against freshly-read content if the file changed underneath the write.
- New dependency: `jsonc-parser` (zero transitive deps), used to edit `tasks.json`/`keybindings.json` via its `modify()`/`applyEdits()` API so a user's existing comments and formatting survive.
- Published to npm; CI (`.github/workflows/publish.yml`) publishes on `v*` tag push or manual dispatch, gated on the full `lint`/`typecheck`/`test`/`build` suite via `prepublishOnly`.

### Fixed

- **Secret screening false-positived on `sourceRef`** — capture's entropy screen was scanning the internal `<resolved path>:<line>` provenance pointer alongside actual fact content. Its generic-token pattern's alphabet includes `/` (needed to catch real base64 secrets, which legitimately contain `/`), so any long forward-slash-delimited path — any Linux/macOS path over ~32 characters, including `mem import --from-md`'s own dedup key — could be blocked as a "high-entropy secret" from directory-name entropy alone, with no secret present. Windows paths never triggered this (`\` isn't in the token alphabet), which is how it stayed latent until Linux CI was wired up. `sourceRef` is no longer screened.
- **`copilot-vscode` wiring tests hardcoded a Windows-only path** — verified against `~/.config/Code/User` (Linux) and `~/Library/Application Support/Code/User` (macOS) in addition to `%AppData%/Roaming/Code/User` (Windows); the installed code itself already branched correctly, only the tests assumed one OS.

## [0.1.0] - 2026-07-13

Initial release: a local-first, correctness-focused long-term memory CLI for AI coding agents (Claude Code, Copilot CLI, Copilot in VS Code, Codex).

### Added

- **Eleven CLI commands** (`mem`, bundled to `dist/token-goat-mem.mjs`): `remember`, `import` (`--from-md`, advisory-only markdown bullet import), `recall`, `list`, `show`, `forget` (soft delete, kept for audit), `pin`, `edit`, `review` (with `--promote`/`--reject` for pending facts, `--summary`/`--section`/`--since-epoch` filters), `epoch` (with `--gc` retention pass: persists contradiction resolutions, prunes superseded facts/sources/audit rows, applies preference decay), and `doctor` (read-only environment/DB health check).
- **SQLite storage** at `~/.mem/mem.db` (override via `TOKEN_GOAT_MEM_HOME`), WAL mode, three tables (`facts`, `sources`, `audit_log` + `meta`), a monotonic write epoch (with a `facts.epoch` column for `--since-epoch` filtering), zero network calls.
- **Anchor-based staleness detection** — nine pure read-only filesystem/git predicates (`file-exists`, `file-absent`, `file-newer-than`, `file-contains`, `file-not-contains`, `newest-of`, `glob-exists`, `git-branch-is`, `git-tracked`) evaluated per recall with three-valued verdicts: `affirmed` / `unverified` / `contradicted`. Only `affirmed` is ground-truth eligible; contradicted facts are excluded from recall and routed to `mem review`. No shell-out (git predicates read `.git/HEAD` and `.git/index` directly), bounded I/O, paths confined to `--root`.
- **Deterministic contradiction resolution** — `subject`+`value` keying; same subject + scope with a different value marks the loser `superseded` (newer + higher-provenance wins), or `contested` and withheld from ground truth when genuinely ambiguous.
- **Secret screening on capture** — named secret patterns plus a generic high-entropy screen; refused facts name a `.mem/allowlist` escape hatch. Suggested (derived) capture, including markdown import, exists as a library seam that can only ever store `pending` facts.
- **BM25 recall with a correctness gate** — relevance ranking, then freshness re-validation, trust filtering, and contradiction/contested exclusion. `--stable` forces deterministic id-sorted output when reproducible ordering matters more than relevance/recency. `--hint-style terse` trims the CTA and shortens kind labels for space-constrained callers. An embedding backend is an injectable seam; no concrete backend ships in v1 (BM25-only in practice).
- **`mem import --from-md <path>`** — advisory-only import: parses a markdown file (CLAUDE.md-style) for `-`/`*` bullet-list lines that look like preference/decision-shaped statements and stores each as a **`pending`, `source_type: "derived"`** fact through the same `captureSuggested` trust path as any other suggested candidate — no code path writes `active`, and there is no bulk-promote shortcut; every imported fact still requires an explicit `mem review --promote <id>`. Bullets nested under an obviously structural heading (e.g. `Architecture`, `File Structure`) are skipped. Provenance (`<absolute file path>:<line>`) is recorded in the fact's `source_ref` field. `--dry-run` reports candidates without writing. Re-importing the same file skips bullets already imported at the same `source_ref` + text.
- **Token-goat seam** — `mem recall --hint-format --root <project-root>` emits the `TGMEM/2` wire format by default: per-fact `display` strings carry no per-line follow-up CTA; instead one shared footer line (`footer  mem show <id> for detail; mem review to resolve contested/pending`) is appended once, when at least one fact line was emitted. `TGMEM/1` (the original per-line-CTA, no-footer format) remains fully supported via `HintFormatOptions.protocolVersion: 1` on the programmatic seam (`buildHintFormat()`); the CLI itself always emits the current default. The seam is one-directional (Mem reads nothing from token-goat), stateless, and self-caveating.
- **Cheap polling via `mem epoch`** — a monotonic integer bumped by every write and left untouched otherwise, so a host tool can skip a full `mem recall` when nothing changed. Plain `mem epoch` never mutates state.
- **Test suite** — 232 vitest tests including fast I/O-free structural guards (`npm run test:guards`) and end-to-end tests against the shipped bundle, isolated from real user data via `TOKEN_GOAT_MEM_HOME`.
- **Docs** — README with a verified copy-paste walkthrough, integration guides for Claude Code / Copilot CLI / Copilot in VS Code / Codex, `AGENTS.md` + `CLAUDE.md`, `SECURITY.md`, `CONTRIBUTING.md`.
